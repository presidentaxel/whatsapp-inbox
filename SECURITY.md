# Politique de sécurité

## Versions prises en charge

Seule la branche principale active du dépôt (`main` ou `master`, selon votre configuration) est considérée comme cible de correctifs de sécurité. Les anciennes branches ou tags ne sont pas maintenus sauf accord explicite de l’équipe.

## Signaler une vulnérabilité

Ne créez pas d’issue publique pour un problème de sécurité sensible (secrets exposés, accès non autorisé aux données, contournement d’auth, etc.).

1. **Dépôt sur GitHub** : utilisez l’onglet [Security advisories](https://docs.github.com/en/code-security/security-advisories) du projet si celui-ci est activé, ou contactez les mainteneurs par un canal privé convenu en interne.
2. **Contenu du rapport** : description du problème, étapes de reproduction, impact estimé, version / commit concerné si vous les connaissez.
3. **Délais** : l’équipe s’efforce de répondre sous **72 heures ouvrées** ; un correctif ou un plan d’action peut nécessiter plus de temps selon la gravité.

Pour les rappels opérationnels (secrets, logs, données personnelles) côté équipe, voir aussi [docs/equipe/securite-conformite.md](docs/equipe/securite-conformite.md).
